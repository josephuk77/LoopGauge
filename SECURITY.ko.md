# 보안 정책

[English](SECURITY.md) | 한국어

LoopGauge는 코딩 에이전트와 프로젝트 명령을 실행하므로 모든 설정과 벤치마크를 실행 가능한 코드처럼 다뤄야 합니다.

## 지원 버전

프로젝트가 실험 단계인 동안 보안 수정은 최신 릴리스와 `main` 브랜치에 적용됩니다.

## 취약점 신고

[GitHub 비공개 취약점 신고](https://github.com/josephuk77/LoopGauge/security/advisories/new)를 이용해 주세요. 자격 증명 노출, 명령 주입, 경로 순회, 공급자 정책 우회 또는 의도하지 않은 네트워크 접근 문제를 공개 이슈로 등록하지 마세요.

개인정보를 제거한 최소 재현 사례를 포함해 주세요. 실제 API 키, 독점 프롬프트, 비공개 저장소 내용 또는 고객 데이터를 포함하면 안 됩니다.

## 운영 경계

- 테스트 권한이 있는 저장소와 환경에서만 최적화를 실행합니다.
- 실행하기 전에 `loop.yaml`의 명령을 검토합니다.
- 권한을 제한한 API 키와 명시적인 공급자별 예산을 사용합니다.
- 작업에 필요하지 않다면 네트워크 접근을 비활성화합니다.
- 생성된 패치와 모델 출력은 검토와 테스트가 끝날 때까지 신뢰하지 않습니다.
