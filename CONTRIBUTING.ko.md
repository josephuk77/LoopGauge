# LoopGauge 기여 안내

[English](CONTRIBUTING.md) | 한국어

LoopGauge는 공급자 어댑터, 결정론적 grader, 가격표 가져오기, 재현 가능한 최적화 루프와 관련된 명확한 이슈와 Pull Request를 환영합니다.

변경 사항을 제출하기 전에 다음 명령을 실행해 주세요.

```bash
npm install
npm run demo
npm run typecheck
npm test
npm run build
```

API를 사용하지 않는 데모는 합성 데이터입니다. 그 값을 실제 모델 성능이나 비용 절감 결과로 소개하면 안 됩니다. 실제 벤치마크 기여는 [벤치마크 방법론](docs/benchmark-methodology.ko.md)을 따라야 합니다.

공급자 중립성은 반드시 지켜야 하는 원칙입니다.

- 핵심 기능에 특정 회사나 모델을 기본 추천으로 추가하지 않습니다.
- 프로젝트 허용 목록에 없는 공급자나 모델을 호출하지 않습니다.
- 공급자별 동작은 어댑터 내부에 격리합니다.
- 단일 공급자 설정과 사용자가 명시적으로 허용한 다중 공급자 설정을 모두 테스트합니다.
- API 키, 숨겨진 추론, 관련 없는 저장소 내용은 로그에 기록하지 않습니다.

새로운 최적화 전략은 전체 비용을 보고하고, 종료 조건을 정의하며, 반복 가능한 평가 근거로 품질을 입증해야 합니다.

처음 기여하기 좋은 항목으로는 결정론적 grader, 비식별화된 벤치마크 fixture, 공급자 가격 가져오기, 리포트 형식, 프로젝트 감지기, 재현 가능한 작업 흐름 문서가 있습니다.
